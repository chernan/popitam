INSTRUCTIONS FOR CREATING A DISTRIBUTION OF POPITAM:  


unzip if necessary  


1) Compiling the tool createDB to index databases  

  • createDB is a tool that indexes any fasta database for Popitam (accepted format is : AC|ID description)

a) use the makefile to compile createDB  

cd POPITAMx.x/CREATE_DB/EXE/ 
make
 

  • the exectuable createDB should be created
  • the usage of createDB is given by typing createDB in the command line (without arguments)

b) run createDB to build a first demo database (using the provided demoDB.faa file):  

./createDB ../testDBs/demoDB_BASE_FORWARDandDECOY.fasta ../testDBs/demoDB_BASE_FORWARDandDECOY.bin demoDB_FORWARDandDECOY NULL NULL BASE_FORWARDandDECOY_1.0 BASE
 

  • the file demoDB_BASE_FORWARDandDECOY.bin is created
  • you can, in the same way, create any database using your own fasta files, as long as the header type is supported

2) Compiling popitam  


a) Now, compile popitam with the following commands:  

cd ../../EXE/ 
make distribue
 


  • you may use the command : make distribue OPTION=o, but if you do so, you'll get some errors later in the perl tests...
    • the directory popitamDist is created; it contains:
    • the executable popitam
    • the parameter file popParam.txt
    • a perl script: testPopitam.pl that can be used to test if the program is doing what expected
    • a text file names : com.txt showing how to run popitam
    • a readme.txt file with some documentation
    • a directory named db with example fasta and indexed databases
    • some stuff (rescoring and auto_popitam_rescored2pidres.pl) that may be used to rescore the results given by Popitam with Phenyx's scoring functions

b) To test the new installation, proceed as follows:  

cd popitamDist 
perl testPopitam.pl
 

  • the test runs two times Popitam with specific example spectra against the demoDB database. If everything goes well, the last line of the test output will be: TEST SUCCESSFUL. In some cases, the test is not successful because of tiny differences in the scores... This is an issue we haven't fixed, but these differences doesn't really affect the interpretation of the output.
